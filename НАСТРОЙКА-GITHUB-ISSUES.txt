НАСТРОЙКА GITHUB ISSUES ДЛЯ НАБЛЮДЕНИЙ

1. СОЗДАНИЕ PERSONAL ACCESS TOKEN
   
   a) Откройте GitHub и перейдите в:
      Settings -> Developer settings -> Personal access tokens -> Tokens (classic)
   
   b) Нажмите "Generate new token (classic)"
   
   c) Задайте название токена (например: "Contour Site Issues")
   
   d) Выберите срок действия (рекомендуется: "No expiration" или длительный срок)
   
   e) Выберите scope (права):
      ✅ repo (полный доступ к репозиторию)
         - Это даст доступ к созданию и чтению issues
   
   f) Нажмите "Generate token"
   
   g) СКОПИРУЙТЕ ТОКЕН СРАЗУ (он больше не будет показан!)
      Вставьте его в assets/config.js в поле GITHUB_TOKEN

2. НАСТРОЙКА РЕПОЗИТОРИЯ
   
   a) Убедитесь, что Issues включены в вашем репозитории:
      Settings -> Features -> ✅ Issues
   
   b) Создайте labels (метки) для модерации:
      - Перейдите в Issues -> Labels
      - Создайте следующие labels:
      
        * "pending" (желтый цвет #fbbf24) - на рассмотрении
        * "unofficial" (синий цвет #60a5fa) - неофициально одобрено
        * "approved" (зеленый цвет #34d399) - подтверждено для досье
        * "rejected" (красный цвет #ef4444) - отклонено

3. НАСТРОЙКА config.js
   
   Откройте assets/config.js и укажите:
   
   GITHUB_REPO: 'imrealstanyxx-maker/contour-archive'
   GITHUB_TOKEN: 'ваш_токен_из_шага_1'
   
   Формат GITHUB_REPO: owner/repo
   Например: 'imrealstanyxx-maker/contour-archive'

4. КАК РАБОТАЕТ МОДЕРАЦИЯ
   
   a) Пользователь создаёт наблюдение на сайте
      → Создаётся issue в GitHub с label "pending"
   
   b) Администратор открывает issue в GitHub
      → Может прочитать наблюдение
      → Может добавить/изменить labels:
         - "unofficial" - неофициально одобрено
         - "approved" - подтверждено для досье
         - "rejected" - отклонено
   
   c) Сайт автоматически читает issues и отображает их
      → Статус определяется по labels
      → "approved" и "unofficial" видны всем
      → "pending" и "rejected" видны только автору и админу

5. ПРОВЕРКА РАБОТЫ
   
   a) Откройте сайт и войдите как observer/admin
   b) Создайте тестовое наблюдение
   c) Проверьте, что issue появился в GitHub Issues
   d) Измените label в GitHub (например, на "approved")
   e) Обновите страницу на сайте - наблюдение должно обновиться

6. БЕЗОПАСНОСТЬ
   
   ⚠️ ВАЖНО: GitHub токен даёт доступ к вашему репозиторию!
   
   - НЕ публикуйте токен в открытом доступе
   - НЕ коммитьте config.js с реальным токеном в публичный репозиторий
   - Используйте GitHub Secrets для GitHub Actions (если нужно)
   - Для публичного сайта рассмотрите использование GitHub App вместо PAT

7. АЛЬТЕРНАТИВА (если не хотите использовать токен в клиенте)
   
   Можно создать простой backend proxy, который будет:
   - Хранить токен на сервере
   - Принимать запросы от сайта
   - Делать запросы к GitHub API
   
   Но для простоты можно использовать токен напрямую в клиенте
   (токен имеет только права на issues, не на весь репозиторий)
