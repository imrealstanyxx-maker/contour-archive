ИНСТРУКЦИЯ: Создание аккаунта администратора "observer"

ШАГ 1: Регистрация аккаунта
1. Откройте сайт на GitHub Pages
2. Нажмите "Регистрация"
3. Заполните форму:
   - Логин: observer
   - Email: ваш_email@example.com (запомните его!)
   - Пароль: придумайте надежный пароль
4. Подтвердите регистрацию через email (если требуется)

ШАГ 2: Установка роли администратора
1. Откройте Supabase Dashboard: https://supabase.com/dashboard
2. Выберите ваш проект
3. Перейдите в раздел "SQL Editor"
4. Откройте файл: supabase/setup-admin-observer.sql
5. Замените 'observer@example.com' на реальный email, который вы использовали при регистрации
6. Выполните SQL-запрос:

UPDATE public.profiles
SET role = 'admin'
WHERE id = (
  SELECT id 
  FROM auth.users 
  WHERE email = 'ВАШ_EMAIL@example.com'  -- ВСТАВЬТЕ ВАШ EMAIL
);

7. Проверьте результат:

SELECT 
  p.id,
  p.username,
  p.role,
  u.email
FROM public.profiles p
JOIN auth.users u ON u.id = p.id
WHERE p.username = 'observer' OR u.email LIKE '%observer%';

Должно показать:
- username: observer
- role: admin
- email: ваш_email@example.com

ШАГ 3: Проверка
1. Выйдите из аккаунта на сайте (если залогинены)
2. Войдите с логином "observer" и вашим паролем
3. Перейдите в "Профиль" - должно быть написано "Роль: Администратор"
4. Переключите доступ на "Внутренний" - должен активироваться внутренний доступ
5. В разделе "Наблюдения" должна появиться кнопка "Модерация"

ГОТОВО! Теперь у вас есть аккаунт администратора "observer".
